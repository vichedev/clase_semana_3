Instalar Uptime Kuma

La herramienta open source para vigilar si tus servicios están "arriba"

Uptime Kuma es un monitor de tiempo de actividad autoalojado, gratuito y muy fácil de usar.

¿Para qué sirve?

🌐

Vigila webs y APIs

HTTP(s), TCP, ping, DNS, puertos…

🔔

Avisa si algo cae

Telegram, email, Discord, WhatsApp…

📈

Historial y uptime

Gráficas de disponibilidad y latencia.

📃

Páginas de estado

Status page pública para tus usuarios.

Requisitos previos

🖥️

Un servidor Linux (ej. el VPS de Hetzner del tema anterior) con Ubuntu/Debian.

👤

Acceso por SSH con un usuario con permisos sudo / root.

🐳

Docker instalado (recomendado) o Node.js 18+ para la instalación manual.

🔌

El puerto 3001 libre (es el que usa Uptime Kuma por defecto).

OPCIÓN A · RECOMENDADA

Instalar Docker

Si aún no tienes Docker en el servidor, instálalo con el script oficial:

# Descargar e instalar Docker
curl -fsSL https://get.docker.com | sh
# Comprobar que funciona
docker --version
OPCIÓN A · PASO A PASO

Levantar Uptime Kuma

# Crear volumen para guardar los datos
docker volume create uptime-kuma
# Iniciar el contenedor
docker run -d \
  --name uptime-kuma \
  --restart=always \
  -p 3001:3001 \
  -v uptime-kuma:/app/data \
  louislam/uptime-kuma:1

--restart=always hace que se reinicie solo si el servidor se apaga.

OPCIÓN B · MANUAL

Instalación con Node.js

Si prefieres no usar Docker, puedes correrlo directamente con Node.js:

git clone https://github.com/louislam/uptime-kuma.git
cd uptime-kuma
npm run setup
# Arrancar (mejor con pm2 para que quede en segundo plano)
node server/server.js

Con pm2: pm2 start server/server.js --name uptime-kuma

Primer acceso

1

Abre el navegador en http://IP-DEL-SERVIDOR:3001

2

En la primera pantalla, crea el usuario administrador (usuario + contraseña).

3

¡Listo! Ya estás dentro del panel de control de Uptime Kuma.

Agregar tu primer monitor

1

Pulsa "Add New Monitor" (+ Añadir monitor).

2

Elige el tipo (HTTP, Ping, TCP…) y pon un nombre.

3

Escribe la URL o IP y el intervalo (ej. cada 60 s).

4

Guarda. Verás en vivo si está UP o DOWN.

Configurar notificaciones

En Settings → Notifications añade un canal y asígnalo a tus monitores:

Buenas prácticas

🔒 Pon HTTPS

Usa un reverse proxy (Nginx / Caddy / NPM) con certificado.

🌍 Usa un dominio

Mejor que entrar por IP:3001; apunta un subdominio.

💾 Respalda los datos

El volumen /app/data guarda toda la config.

⬆️ Actualiza

Baja la nueva imagen y recrea el contenedor.

✅

Resumen del tema

Uptime Kuma monitorea webs, APIs y servidores.
La forma más fácil de instalarlo es con Docker.
Se accede en el puerto 3001 y creas un admin.
